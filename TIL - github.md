## Git으로 할 수 있는 것

- `git`
  1.  프로젝트의 버전 관리를 위한 **도구**
      1. ‘최종\_최종.ppt’란 게 필요 없어지고 **하나의 파일로 버전관리를 할 수 있게 도와준다** : history [(ex. 헌법 개정)](https://github.com/ohahohah/constitution-of-republic-of-korea/commits/main)
  2.  작업 단위 나누기
      1. 기능을 완성할 때마다 작업 내용을 저장하여, 추후 발생하는 **_에러 관리 가능_**
  3.  협업해서 하나의 프로젝트를 만드는데 유용

      1. 여러 사람이 프로젝트를 나누어서 하나로 합치는 일이 편함
         - 언제, 누가, 어떤 부분을 수정했는지 파악 가능
      2. Git을 사용하지 않고 덮어쓰는 형태로 관리한다면, 다른 사람이 작업한 내용을 내 파일로 덮어쓸 수 있다. **_(에러 관리 불가능)_**
      3. Git을 사용하면 같은 파일명의 내용이 어떤 부분이 다른지 자동 비교 및 선택 반영 가능

      <aside>
      🍇 **자동 비교 되는 파일**

      1. 코드(Python, HTML, JavaScript, Java … )
      2. text 파일
      3. markdown 파일
      4. CSV 파일

      ***

      이미지, Word, PDF, 엑셀은 여러가지 설정을 해주어야 가능

      </aside>

## Github으로 할 수 있는 것

- `github` : git 원격 저장소 + git으로 할 수 있는 커뮤니티 기능 서비스(원격 클라우드)
- `sourcetree` : git을 쉽게 사용할 수 있는 도구

  1. 전 세계에서 가장 많이 사용되는 git 도구 중 하나
  2. 초심자 ~ 중급자 여러가지 기능 제공
  3. git 사용 패턴에 집중하기 위해서 사용
     - git 사용패턴을 이해하고 익숙해지는 것 (not 사용법 암기)

- git은 여러가지 방법으로 사용할 수 있다. sourcetree 이외 터미널에 명령어를 직접 입력할수도 있다. (CLI 방법)

## 버전관리와 commit - 개념

- 중간중간 git을 이용해 현재 프로젝트의 상태를 저장해주기 (컨트롤 + S 처럼)
- `누가, 언제, 현재 프로젝트의 상태가 어떤지(현재 파일의 내용들)` 세 가지 정보를 포함해 작업 내용을 관리
- `commit` : ‘현재 프로젝트 상태’를 저장한 것
  - git은 commit(커밋)을 통해 ‘현재 프로젝트의 상태’를 저장하고 조회
  - snapshot처럼 현재 프로젝트의 **전체 상태를** **포착하는 것**
  - 커밋들은 언제 했는지 정보도 포함하고 있으므로 순서대로 보면 그 자체로 히스토리라고 할 수 있다. **commit history, commit log(로그, 기록)**
